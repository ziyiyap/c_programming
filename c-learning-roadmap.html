Week 01
C Foundations — Hello, Low Level
Compilation, types, variables, I/O, operators
+
🐍 Coming from Python

Python runs scripts interpreted. C is compiled — you run gcc to turn source code into a binary. There's no REPL; every program needs a main() function.

Python
name = "Alice"
age = 30
print(f"Hello, {name}!")
C
#include <stdio.h>
int main() {
    char name[] = "Alice";
    int age = 30;
    printf("Hello, %s!\n", name);
    return 0;
}
📖 Core Concepts
  • Data types: int, float, double, char, long, unsigned int — sizes are fixed and matter
  • Compilation pipeline: Source → Preprocessor → Compiler → Assembler → Linker → Binary
  • printf / scanf: Formatted I/O with format specifiers (%d, %f, %s, %c)
  • Operators: Arithmetic, relational, logical, bitwise (&, |, ^, ~, <<, >>)
  • Type casting: Explicit vs implicit conversion
⚙️ Compile & Run
# Install GCC (Linux/Mac)
sudo apt install gcc   # Ubuntu
brew install gcc       # Mac

# Compile and run
gcc -Wall -o hello hello.c
./hello

# -Wall turns on all warnings — always use this!

  2. Compute and print the size of each data type using sizeof()
  3. Build a temperature converter: Celsius ↔ Fahrenheit
  4. Use bitwise operators to toggle, set, and clear individual bits of an integer

Week 02
Control Flow & Functions
if/else, loops, switch, functions, scope, recursion
+
🐍 Coming from Python

Python uses indentation for blocks; C uses { }. C's for loop is much more flexible — the loop variable can be mutated inside. There's no Python-style for x in list — you iterate with indices.

Python
for i in range(10):
    if i % 2 == 0:
        print(i)
C
for (int i = 0; i < 10; i++) {
    if (i % 2 == 0) {
        printf("%d\n", i);
    }
}
📖 Core Concepts
  • if / else if / else — same logic as Python, different syntax
  • switch / case — efficient multi-branch for integers/chars; don't forget break!
  • while / do-while / for — C's three loop types
  • Functions: Return type, parameters — everything must be typed. Declare before use (or use a prototype)
  • Scope: Variables live in their block { }. No closures, no global-by-default
  • Recursion: C supports recursion but has a finite stack — deep recursion causes stack overflow
💡 Function Example
// Function prototype (declaration)
int factorial(int n);

int main() {
    printf("%d\n", factorial(5)); // 120
    return 0;
}

// Function definition
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}
🏋️ Exercises
  1. FizzBuzz in C (classic, but notice the type discipline)
  2. Write isPrime(int n) — returns 1 if prime, 0 otherwise
  3. Fibonacci sequence iteratively and recursively — compare performance
  4. Write a menu-driven program using switch for a unit converter

Number Guessing Game: Computer picks a random number (use rand() from stdlib.h). Player guesses with hints "higher/lower". Count and display attempts. Replay option using a loop.

Week 03
Arrays & Strings
Fixed arrays, C-strings, string.h, multi-dimensional arrays
+
🐍 Coming from Python

Python lists resize dynamically and hold mixed types. C arrays are fixed-size and typed, stored in contiguous memory. Strings in C are arrays of char ending with a null terminator '\0' — there's no built-in string type!

Python
nums = [1, 2, 3, 4, 5]
msg = "Hello"
print(len(msg))  # 5
C
int nums[5] = {1,2,3,4,5};
char msg[] = "Hello"; // 6 bytes incl \0
printf("%zu\n", strlen(msg)); // 5
📖 Core Concepts
  • Array declaration: int arr[10]; — uninitialized arrays contain garbage!
  • Array bounds: C does NOT check bounds — writing past the end is undefined behavior
  • C-strings: Null-terminated char arrays. Functions: strlen, strcpy, strcat, strcmp, sprintf
  • Multi-dimensional arrays: int grid[3][4]; stored row-major
  • Array ↔ pointer duality: arr is essentially a pointer to arr[0] (previews week 4)
⚠️ Critical Warning
⚠️ Buffer Overflow

Never use gets() — it's been removed from C11. Use fgets(buf, sizeof(buf), stdin) instead. Always ensure your destination buffer is large enough before copying strings.

🏋️ Exercises
  1. Implement bubble sort and selection sort on an integer array
  2. Write myStrlen, myStrcpy, myStrcat from scratch
  3. Count vowels, consonants, and spaces in a string
  4. Transpose a 3×3 matrix stored as a 2D array
📦 Mini Project — Week 3

Student Grade Manager: Store names (char arrays) and grades (float array) for N students. Compute average, find highest/lowest scorer, print a sorted leaderboard.

Week 04
Pointers — The Heart of C
Pointer arithmetic, pointer to pointer, function pointers
+
🐍 Coming from Python

In Python, everything is an object reference — you never see addresses. In C, a pointer is a variable that holds a memory address. This is the concept that trips most beginners but unlocks everything — hardware access, dynamic memory, efficient data structures, and OS interfaces.

📖 Core Concepts
int x = 42;
int *p = &x;      // p holds the address of x

printf("%p\n", p);    // prints address e.g. 0x7ffd...
printf("%d\n", *p);   // dereference: prints 42

*p = 100;  // modifies x through the pointer!
printf("%d\n", x);   // prints 100
  • & (address-of): Gets the memory address of a variable
  • * (dereference): Follows the pointer to read/write the value it points to
  • Pointer arithmetic: p + 1 moves forward by sizeof(*p) bytes — crucial for arrays
  • NULL pointer: Always initialize pointers; uninitialized pointers cause crashes
  • Pointer to pointer: int **pp — used for modifying pointers in functions, 2D arrays
  • Function pointers: int (*fn)(int, int) — enables callbacks, dispatch tables (key in embedded/robotics)
💡 Pass by Reference (unlike Python's default)
// Without pointer: x is NOT modified
void badSwap(int a, int b) { int t=a; a=b; b=t; }

// With pointer: x IS modified
void swap(int *a, int *b) {
    int t = *a;
    *a = *b;
    *b = t;
}
// Call it:
swap(&x, &y);
🏋️ Exercises
  1. Implement swap(int *a, int *b) and verify with prints
  2. Traverse an array using pointer arithmetic (no [] syntax)
  3. Write a function that takes a char * and reverses the string in-place
  4. Create a function pointer array to dispatch arithmetic operations
📦 Mini Project — Week 4

Sorting with Function Pointers: Implement a generic sort() that accepts an array and a comparator function pointer, so callers can sort ascending, descending, or by custom criteria.

Week 05
Dynamic Memory Management
malloc, calloc, realloc, free, memory leaks, valgrind
+
🐍 Coming from Python

Python manages memory for you via garbage collection. In C, you are the memory manager. You allocate memory on the heap with malloc, and you must free it when done. Forgetting to free causes memory leaks; freeing twice or using freed memory causes crashes. This is hard but powerful — it's why C is used in embedded and real-time systems.

📖 Core Concepts
#include <stdlib.h>

// Allocate array of 10 ints on the heap
int *arr = (int*)malloc(10 * sizeof(int));
if (arr == NULL) { /* always check! */ }

// calloc: allocates AND zeroes memory
int *zeros = (int*)calloc(10, sizeof(int));

// realloc: resize an existing allocation
arr = (int*)realloc(arr, 20 * sizeof(int));

// free: ALWAYS free when done
free(arr);
arr = NULL; // good practice: avoid dangling pointer
  • Stack vs Heap: Local variables are on the stack (fast, limited, auto-freed). malloc allocates on the heap (slower, large, manual)
  • Memory leak: Allocated memory that's never freed. In robotics/embedded, this kills long-running programs
  • Dangling pointer: Pointer that references freed memory — accessing it is undefined behavior
🔍 Debug with Valgrind
# Install
sudo apt install valgrind

# Run your program through valgrind
valgrind --leak-check=full ./myprogram

# Output will show memory leaks, invalid reads/writes
🏋️ Exercises
  1. Implement a dynamic integer array that doubles in capacity when full
  2. Write a function that duplicates a string on the heap (like strdup)
  3. Create a 2D dynamic matrix (pointer to pointer) with variable rows/cols
  4. Run valgrind on your week 3 project and fix any issues
📦 Mini Project — Week 5

Dynamic Linked List: Build a singly linked list with append, prepend, delete, search, and print operations. Free all nodes properly on exit. Verify with valgrind.

Week 06
Structs & Custom Data Types
struct, typedef, enum, nested structs, struct pointers
+
🐍 Coming from Python

Python's class bundles data and methods together. C's struct bundles only data — no built-in methods (but you can achieve OOP-like patterns using function pointers!). This is the backbone of all complex C programs.

Python
class Robot:
    def __init__(self, name, x, y):
        self.name = name
        self.x = x
        self.y = y
C
typedef struct {
    char name[32];
    float x, y;
} Robot;

Robot r1 = {"R2D2", 0.0, 0.0};
📖 Core Concepts
// Arrow operator for pointer-to-struct
Robot *rp = &r1;
printf("%s at (%.1f, %.1f)\n",
       rp->name, rp->x, rp->y);

// Equivalent to: (*rp).name

// Enum for readable constants
typedef enum { IDLE, MOVING, ERROR } RobotState;

// Nested struct
typedef struct {
    Robot body;
    RobotState state;
    float battery;
} FullRobot;
  • typedef: Creates an alias — typedef struct Point { ... } Point; lets you write Point p instead of struct Point p
  • Arrow (->) vs dot (.): Use . for struct values, -> for struct pointers
  • Struct padding: The compiler may add padding bytes for alignment — use sizeof() to check
🏋️ Exercises
  1. Define a Point struct and write functions to compute distance between two points
  2. Create a Stack struct backed by a dynamic array with push/pop/peek
  3. Model a simple Sensor struct with type (enum), value, and timestamp
  4. Build a "polymorphic" shape (Circle, Rectangle) using function pointers in a struct
📦 Mini Project — Week 6

Week 07
File I/O & the Preprocessor
fopen, fread, fwrite, #define, #include, header files, Makefiles
+
📖 Core Concepts — File I/O
#include <stdio.h>

// Open for writing
FILE *f = fopen("log.txt", "w");
if (!f) { perror("fopen"); return 1; }

fprintf(f, "Robot at %.2f, %.2f\n", x, y);
fclose(f);

// Read line by line
FILE *r = fopen("data.csv", "r");
char line[256];
while (fgets(line, sizeof(line), r)) {
    // process line
}
fclose(r);
  • Mode strings: "r" read, "w" write (truncate), "a" append, "rb"/"wb" binary
  • fread/fwrite: Read/write raw binary data — essential for sensor data logging
  • fseek/ftell: Navigate within files
📖 Headers & Multi-file Programs
// robot.h — declarations only
#ifndef ROBOT_H
#define ROBOT_H

typedef struct { float x, y; } Robot;
void moveRobot(Robot *r, float dx, float dy);

#endif

// robot.c — definitions
#include "robot.h"
void moveRobot(Robot *r, float dx, float dy) {
    r->x += dx; r->y += dy;
}
🔨 Basic Makefile
CC = gcc
CFLAGS = -Wall -g

all: robot_sim

robot_sim: main.o robot.o
	$(CC) $(CFLAGS) -o robot_sim main.o robot.o

main.o: main.c robot.h
	$(CC) $(CFLAGS) -c main.c

clean:
	rm -f *.o robot_sim

Week 08
Systems Programming & POSIX
Processes, signals, pipes, threads (pthreads), timers
+
📖 Core Concepts
  • Processes: fork(), exec(), wait() — create and manage child processes
  • Signals: SIGINT, SIGTERM, SIGUSR1 — async notifications between processes. Use signal() or sigaction()
  • Pipes: pipe(), read(), write() — IPC between processes. Foundation of shell piping
  • POSIX threads: pthread_create, pthread_join, pthread_mutex_lock — concurrent execution

#include <pthread.h>

void* sensorThread(void *arg) {
    while (1) {
        readSensor();
        usleep(10000); // 10ms
    }
    return NULL;
}

// In main:
pthread_t tid;
pthread_create(&tid, NULL, sensorThread, NULL);
// compile with: gcc -lpthread

Tools & IDEs

Everything you need to write, compile, debug, and profile C programs.

🔨

GCC (GNU Compiler)

The standard C compiler on Linux/Mac. Always compile with gcc -Wall -Wextra -g. The -g flag adds debug symbols for GDB. Available on Windows via MinGW or WSL.

🍎

Clang (macOS)

Apple's default C compiler. Often gives better error messages than GCC. Comes pre-installed on macOS via Xcode Command Line Tools. clang -Wall -g

🐛

GDB — GNU Debugger

Step through code line by line, set breakpoints, inspect variables and memory. Essential skill. Basic commands: break main, run, next, print x, backtrace

🔍

Valgrind

Detects memory leaks, invalid memory accesses, use-after-free, and double-free errors. Run valgrind --leak-check=full ./prog. Linux-only (use WSL on Windows).

📝

VS Code + C/C++ Extension

Best general-purpose IDE for beginners. Install the "C/C++" extension by Microsoft. Configure tasks.json for build and launch.json for GDB debugging with a UI.

🖥

CLion (JetBrains)

Full-featured C/C++ IDE with excellent refactoring, debugger integration, and Valgrind plugin. Free for students via JetBrains Education. Best professional option.

🌐

Compiler Explorer (godbolt.org)

Paste C code and instantly see the assembly output. Essential for understanding what the compiler does. Great for comparing -O0 vs -O2 optimizations.

⚡

AddressSanitizer (ASan)

Compile with -fsanitize=address to detect memory bugs at runtime with minimal overhead. Much faster than Valgrind. Use both: ASan for fast iteration, Valgrind for thorough checks.

📊

gprof / perf

Profile your C programs to find bottlenecks. gprof: compile with -pg, run program, then gprof ./prog gmon.out. perf: Linux-native, extremely powerful for systems work.

🔄

Make / CMake

make for simple projects (learn this first). cmake for multi-platform projects — required by ROS and many robotics frameworks. Learn Makefiles in week 7.

🐧

WSL2 (Windows Only)

If on Windows, use Windows Subsystem for Linux 2. Gets you a real Linux environment with GCC, GDB, Valgrind, pthreads, and all POSIX APIs. Strongly recommended over Cygwin or MinGW.

🧹

clang-format & clang-tidy

Auto-format your code (clang-format) and get lint warnings about risky patterns (clang-tidy). Integrate into VS Code for format-on-save. Builds good habits early.

Common Beginner Mistakes

These are the pitfalls that waste hours. Read them now; recognize them in the wild.

1. Forgetting to Initialize Variables

In Python, uninitialized variables throw NameError. In C, int x; gives x a garbage value — random bits already in memory. Always initialize: int x = 0;, and especially int arr[10] = {0}; for arrays.

2. Off-by-One in Arrays

C arrays are 0-indexed and bounds are NOT checked. Accessing arr[10] on an array of size 10 is undefined behavior — it might crash, corrupt memory, or silently do the wrong thing. The worst bugs say nothing; they just produce wrong output.

3. Using = Instead of == in Conditions

if (x = 5) assigns 5 to x (always true!). if (x == 5) compares. Pro trick: write if (5 == x) — if you accidentally use =, the compiler will error because you can't assign to a literal.

4. Missing break in switch Statements

C switch cases fall through by default. Without break, execution continues into the next case. This is occasionally useful but almost always a bug. Always add break unless fall-through is intentional and commented.

5. Returning Pointers to Local Variables

return &localVar; is a dangling pointer — the variable is destroyed when the function returns. The memory is then reused. Use malloc to allocate on the heap (and free it later), or pass a pointer as a parameter for the function to fill.

6. Memory Leaks & Double Free

Every malloc needs exactly one free. Forgetting leaks memory. Freeing twice corrupts the heap allocator and causes crashes. Set pointers to NULL after freeing: free(p); p = NULL;. Use Valgrind to catch these.

7. Using gets() or scanf("%s") for Strings

gets() has no bounds checking and was removed from C11. scanf("%s") also doesn't limit input length. Use fgets(buf, sizeof(buf), stdin) or scanf("%255s", buf) with an explicit limit.

8. Integer Overflow (Silent!)

A signed int on most systems is 32-bit: max value ~2.1 billion. 2147483647 + 1 wraps to a negative number with no warning or error. Use long long or uint64_t for large values, and be careful with multiplication before division.

9. Comparing Strings with ==

From Python: if (name == "Alice"). In C, this compares pointer addresses, not string content — it's almost always false (or coincidentally true). Use strcmp(name, "Alice") == 0 instead.

10. Ignoring Compiler Warnings

Warnings in C are often errors-in-waiting. Always compile with -Wall -Wextra and treat warnings as errors with -Werror during learning. A warning about an implicit conversion or unused variable is the compiler telling you something is wrong.

Debugging & Efficiency Tips

How to debug fast, build good habits, and accelerate your learning.

GDB Survival Cheat Sheet

# Compile with debug symbols
gcc -g -o myprog myprog.c

# Launch GDB
gdb ./myprog

# Key commands inside GDB:
(gdb) break main       # set breakpoint at main
(gdb) break 42         # set breakpoint at line 42
(gdb) run              # start the program
(gdb) next             # step over (n)
(gdb) step             # step into function (s)
(gdb) print x          # print variable x
(gdb) print *ptr       # dereference pointer
(gdb) backtrace        # show call stack (bt)
(gdb) watch x          # break when x changes
(gdb) continue         # resume until next break (c)
(gdb) quit             # exit GDB

Compile Flags That Save You

# The flags to always use during development:
gcc -Wall -Wextra -g -fsanitize=address,undefined \
    -o myprog myprog.c

# -Wall        : all common warnings
# -Wextra      : extra warnings
# -g           : debug symbols for GDB
# -fsanitize=address   : detect memory bugs at runtime
# -fsanitize=undefined : detect UB (signed overflow, etc.)

# For release (optimized, no debug):
gcc -O2 -o myprog myprog.c

Debugging Strategy

Use this process when stuck:

  1. Read the error message top-to-bottom — the first error is often the real one
  2. Isolate: Comment out code until the bug disappears — last removed block is suspect
  3. Add printf breadcrumbs — print variable values at key points
  4. Use GDB backtrace on a crash — it tells you exactly where and what was on the call stack
  5. Run under Valgrind for memory-related crashes
  6. Check assumptions: Are arrays the size you think? Are pointers NULL? Did malloc return NULL?
  7. Rubber duck: Explain the code out loud line by line — bugs often reveal themselves

Learning Efficiently

  • Code every day — even 20 minutes beats 3-hour marathon sessions twice a week
  • Type code; don't copy-paste — muscle memory and typo debugging are part of learning
  • Read other people's C — study Redis, SQLite, or the Linux kernel source on GitHub
  • Draw memory diagrams — for pointers and structs, draw boxes and arrows on paper
  • Build something you care about — motivation matters; adapt exercises to robotics themes
  • Use Compiler Explorer (godbolt.org) to see what your C becomes in assembly
  • Read the man pages — man 3 malloc, man 3 fopen — learn to read primary docs
  • Commit to Git — even personal projects. Easy to rollback broken experiments

Python → C Mental Model Shifts

Python thinking
  • Variables hold objects
  • Strings are objects
  • Lists resize automatically
  • GC frees memory
  • Everything is a reference
  • Bounds checked
C thinking
  • Variables hold values/bits
  • Strings are char arrays + \0
  • Arrays are fixed; use malloc to grow
  • You free memory manually
  • Pointers are explicit addresses
  • You own bounds checking
